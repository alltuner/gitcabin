URL still pushes via history.pushState (default). hx-target/select/swap swap only
rather than the whole body, so the header doesn't flicker. We GET the new page, pick out its
, and outerHTML-swap ours. show:window:top scroll to top after the swap, mimicking what a real navigation would do. preload + preload="mouseover" on body opts every descendant into the preload extension. htmx-ext-preload's getClosestAttribute walk inherits the value down without per-link annotation. When the user clicks ~200 ms later, htmx's GET under hx-boost reads from cache and the swap is essentially free. #}
