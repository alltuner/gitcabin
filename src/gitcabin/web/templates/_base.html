The wordmark "gitcabin" carries the project's git identity; the mark just needs to be a cabin. #} gitcabin {# Theme switcher: three radios sharing name="theme". The light + dark radios carry DaisyUI's `theme-controller` class, so when one is checked, `:root:has(input.theme-controller[value=...]:checked)` fires and DaisyUI swaps its palette variables — entirely in CSS, no JS. The "system" radio has no theme-controller class, so checking it leaves no theme override and DaisyUI's --prefersdark modifier follows OS pref naturally. main.ts only persists the choice in localStorage and pre-checks the right radio at boot — the visual switch itself is browser-native.
